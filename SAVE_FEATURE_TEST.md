# 提供商配置保存功能测试指南

## 功能概述
在"提供商配置"标签页中添加了保存功能，用户可以保存所有提供商配置到浏览器的localStorage中。

## 测试步骤

### 1. 访问应用
- 打开浏览器访问 http://localhost:5173
- 应该能看到4个标签页：模型对比测试、国内模型专区、风水图片分析、提供商配置

### 2. 测试保存功能
1. 点击第4个标签"提供商配置"
2. 修改任意提供商的配置，例如：
   - 修改API Key
   - 修改模型名称
   - 修改系统提示词
   - 调整温度参数
3. 点击"保存所有配置"按钮
4. 观察是否显示"配置已保存成功！"的提示信息

### 3. 验证持久化
1. 修改配置后点击保存
2. 刷新浏览器页面
3. 返回"提供商配置"标签页
4. 检查之前的修改是否仍然保持

### 4. 测试添加新提供商
1. 点击"添加提供商"按钮
2. 填写新提供商的信息
3. 点击"保存所有配置"按钮
4. 刷新页面，检查新提供商是否仍然存在

### 5. 测试删除提供商
1. 展开任意提供商的折叠面板
2. 点击删除图标
3. 点击"保存所有配置"按钮
4. 刷新页面，检查该提供商是否已被删除

## 技术实现

### 保存机制
- 使用Zustand状态管理库的`saveProviders`方法
- 将配置序列化为JSON字符串存储在localStorage中
- 使用localStorage键名：`airooter_providers`

### 自动保存
- 添加新提供商时自动保存
- 删除提供商时自动保存
- 手动点击"保存所有配置"按钮进行完整保存

### 错误处理
- 保存失败时会显示错误信息
- 所有操作都有3秒的状态提示

## 调试工具

### 1. 浏览器开发者工具
- 打开F12开发者工具
- 切换到Application/Storage标签
- 查看LocalStorage中的`airooter_providers`键值

### 2. 调试页面
- 打开`debug-localstorage.html`文件
- 可以查看、清除和设置测试数据

## 预期行为

✅ **正常情况：**
- 点击保存按钮显示成功提示
- 刷新页面后配置保持不变
- 所有提供商配置正确加载

❌ **错误情况：**
- localStorage不可用时显示错误
- JSON序列化失败时显示错误
- 存储空间不足时显示错误
